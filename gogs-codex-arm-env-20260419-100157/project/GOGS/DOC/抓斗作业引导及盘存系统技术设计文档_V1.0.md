### 3.2 硬件安装与连接

#### 3.2.1 雷达安装
- **安装位置**：行车转盘下方或侧面，确保水平360°无遮挡。雷达中心与转盘旋转中心需精确测量偏移量。
- **安装角度**：建议向下倾斜5°~10°，使0°水平线对准料堆中部。
- **电气连接**：网线连接至交换机，供电12V DC（由服务器或独立电源提供）。

#### 3.2.2 球机安装
- **安装位置**：与雷达同轴或附近，视野覆盖抓斗作业区域。安装高度应高于抓斗最高点。
- **电气连接**：网线至交换机，支持POE供电或独立12V/24V。

#### 3.2.3 边缘服务器安装
- **安装位置**：行车电气室或现场控制柜，注意散热，避免阳光直射。
- **网络连接**：
  - 网口1：连接雷达、球机，配置独立子网（如192.168.111.x/24）。
  - 网口2：连接企业监控网络，可配置DHCP或静态IP。
- **串口扩展**：通过USB转RS485模块连接PLC（若PLC支持Modbus RTU）。
- **存储挂载**：2TB SSD 格式化为 ext4，挂载至 `/data` 或项目运行目录，用于视频录像、点云快照、日志和数据库备份持久化。

#### 3.2.4 与PLC连接
- 边缘服务器通过Modbus TCP（以太网）或Modbus RTU（RS485）与行车PLC通信，获取行车位姿并发送引导坐标。优先使用TCP，若PLC不支持则用RTU。

  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      // 尝试刷新Token
      const refreshToken = localStorage.getItem('refreshToken')
      if (refreshToken) {
        const response = await axios.post('/auth/refresh', { refresh_token: refreshToken })
        if (response.data.success) {
          localStorage.setItem('token', response.data.token)
          // 重试原请求
          error.config.headers.Authorization = `Bearer ${response.data.token}`
          return apiClient(error.config)
        }
      }
      // 刷新失败，跳转登录
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }

## 6. 核心算法设计

### 6.1 雷达极坐标转直角坐标

根据雷达说明书，转换公式如下：

```
X = r * cos(ω) * cos(-α) + R * cos(-α)
Y = r * cos(ω) * sin(-α) + R * sin(-α)
Z = r * sin(ω) + z
```

- r：实测距离（米）
- ω：垂直角（弧度，范围 -5°~+45°）
- α：水平角（弧度，范围 0~2π）
- R：接收光心到原点的平面半径（设备参数）
- z：光心到原点的Z轴高度（设备参数）

### 6.2 坐标系定义与变换

#### 6.2.1 坐标系定义
| 符号 | 名称 | 原点 | 轴向 |
|------|------|------|------|
| R系 | 雷达坐标系 | 雷达底座中心 | X前，Y左，Z上 |
| T系 | 转盘坐标系 | 转盘旋转中心 | 与R系平行 |
| G系 | 大车坐标系 | 大车固定点 | X沿轨道，Y垂直轨道，Z上 |
| W系 | 世界坐标系 | 料场固定原点 | 同G系 |

#### 6.2.2 变换公式
1. **R → T**：`p_t = p_r + t_r2t`
2. **T → G**：`p_g = R_z(-θ) * p_t`
3. **G → W**：`p_w = p_g + (X_g, Y_g, 0)`

### 6.3 最高点检测算法

输入：G系点云（经过下采样）
输出：最高点坐标 (X_max, Y_max, Z_max) 在G系，同时计算W系坐标。

步骤：
1. **ROI裁剪**：以当前抓斗位置为中心，设定矩形区域
2. **统计滤波**：使用 `pcl::StatisticalOutlierRemoval`，邻近点数50，标准差倍数1.0
3. **体素下采样**：使用 `pcl::VoxelGrid`，体素边长0.05m
4. **搜索最高点**：遍历点云，记录Z值最大的点
5. **平滑**：保留最近5帧的最高点，取平均值
6. **计算绝对坐标**：将相对最高点变换到W系

### 6.4 体积计算算法

输入：全局地图、货场边界多边形
输出：各料堆体积列表

步骤：
1. **边界裁剪**：使用 `pcl::CropHull` 提取边界内点云
2. **地面拟合**：RANSAC平面拟合或高度直方图法
3. **料堆分割**：欧式聚类，聚类阈值0.5m
4. **体素积分**：计算料堆体积
5. **结果输出**：料堆编号、体积、测量时间、置信度

## 8. 核心处理模块详细设计

### 8.1 雷达驱动与解析模块

- **类名**：`LidarDriver`
- **职责**：
  - 创建UDP socket，绑定端口5600
  - 接收数据包，解析协议，生成PCL点云
  - 将点云放入环形缓冲区

### 8.2 坐标转换模块

- **类名**：`CoordinateTransformer`
- **输入**：雷达点云（R系）、当前位姿（X_g, Y_g, θ）
- **输出**：大车坐标系点云、世界坐标系点云

### 8.3 最高点检测模块

- **类名**：`HighestPointDetector`
- **输入**：大车坐标系点云
- **输出**：最高点坐标（G系和W系）
- **平滑**：`std::deque<Point>` 保存最近5帧最高点

### 8.4 体积计算模块

- **类名**：`VolumeCalculator`
- **方法**：`std::vector<PileVolume> calculate(PointCloudPtr map, const Polygon& boundary)`

### 8.5 Modbus通信模块

- **类名**：`ModbusClient`
- **基于**：Qt SerialBus（`QModbusTcpClient` / `QModbusRtuSerialClient`）
- **读取频率**：50ms周期（20Hz）
- **写入频率**：100ms周期（10Hz）
