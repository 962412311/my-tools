# 海康设备联调 SOP

## 目标

这份文档用于把海康摄像机在当前项目里的接入、联调、排障和回写顺序固定下来。

适用范围：

- 海康球机或普通网络摄像机
- 固定现场 `ARM Linux + Qt 后端 + Vue 前端`
- 当前项目视频主线：`RTSP + MediaMTX + WebRTC/WHEP + HLS + ONVIF + GStreamer`

## 一、联调前准备

联调前至少确认这些信息已经拿到：

- 相机 IP
- `RTSP` 地址
- `ONVIF` 账号密码
- 是否要求 `WS-Security`
- 主码流编码、子码流编码
- 主码流分辨率、子码流分辨率
- 主码流帧率、子码流帧率
- 目标浏览器和版本

建议默认基线：

- 主码流：录像优先，默认 `2K`
- 子码流：预览优先，默认 `720p`
- 子码流编码：优先 `H.264`
- 主码流编码：按录像链路在 `H.264 / H.265` 间确认

## 二、固定联调顺序

不要跳步骤，固定按这个顺序来：

1. 配置核对
2. 实时预览
3. 抓图
4. 录像
5. 回放
6. PTZ
7. 回写结论

## 三、配置核对

先看配置页和监控页，不要先点动作：

1. 确认 `camera/main_stream_* / sub_stream_*` 与现场相机设置一致
2. 确认 `recording/input_codec` 是否符合现场编码
3. 确认监控页顶部“视频基线”与配置页一致
4. 确认监控页顶部“配置源”显示已同步

如果这一步都对不上，后面的截图、录像、回放结论都不可信。

## 四、实时预览

进入监控页后，固定按这个顺序判断：

1. 看“实时链路”
2. 看“链路异常”
3. 看“MediaMTX”
4. 看“端口探测”
5. 看“浏览器兼容”
6. 看“视频自检”

通过标准：

- 默认优先命中 `WebRTC`
- 需要降级时能明确显示 `HLS`
- 若 `MediaMTX` 或端口异常，页面能直接给出诊断

## 五、抓图

抓图固定按两层验证：

1. 先验证前端本地截图
2. 再验证后端 `/api/video/snapshot` 兜底

通过标准：

- 本地截图成功，或
- 本地失败后后端兜底成功

如果两层都失败，优先回写：

- 当前浏览器
- 当前链路
- 当前编码
- `/api/video/snapshot` 真实错误

## 六、录像与回放

先看状态，再按动作验证：

1. 看“录像前提”
2. 看“录像健康”
3. 看“插件 / 缺失插件”
4. 看“录像编码”
5. 看“保留策略”

动作验证顺序：

1. `POST /api/video/recording/start`
2. 观察 `healthStatus / recoveryStatus`
3. `POST /api/video/recording/stop`
4. 确认录制文件可播放
5. 在回放页确认文件可回放

通过标准：

- 能正常开始录像
- 停止后能得到有效录像文件
- 文件能播放、下载、回放
- 若进程异常退出，自动恢复状态有留痕

## 七、PTZ

PTZ 固定按这个顺序，不要一上来就连续移动：

1. 看 `/api/video/ptz/status`
2. 做 `stop`
3. 做 `presets` 读取
4. 做 `goto preset`
5. 做 `move`
6. 做 `relative-move`
7. 最后测 `home`

重点确认：

- `configured / ready / rtspConfigured / onvifConfigured / usernameConfigured`
- `wsSecurityEnabled`
- 最近错误和最近动作
- 前端方向与实际动作是否一致

## 八、现场必须回写的文档

联调结束后必须同步更新：

1. [`视频实机验收矩阵模板`](视频实机验收矩阵模板.md)
2. [`现场联调验收记录模板`](现场联调验收记录模板.md)
3. [`视频现场回写顺序卡`](视频现场回写顺序卡.md)
4. `todo.md`

至少回写这些结论：

- 默认实时链路是否稳定维持在 `WebRTC`
- 哪些浏览器或终端必须退回 `H.264`
- 哪些浏览器或终端可以直放 `H.265`
- 当前相机机型的 `recording/input_codec` 最终建议
- 当前机型的 PTZ / Snapshot / Recording 是否都通过

## 九、问题分类回写

发现问题时，不要只写“失败”，按下面分类回写：

- 实时链路问题：写到 `视频链路与ONVIFREADME.md`
- 浏览器兼容问题：写到 `视频浏览器兼容矩阵.md`
- 海康设备差异问题：写到 `海康SDK与ISAPI能力边界.md`
- 通用现场步骤问题：写到本 SOP
- 状态与优先级问题：写到 `todo.md`

## 十、当前已知约束

- 当前还没有真实摄像头环境回写完 `H.264 / H.265` 实机矩阵
- 当前 `camera/main_stream_* / sub_stream_*` 只用于联调基线，不会自动下发设备
- 当前不把海康 `SDK` 当成默认播放链
- 当前 `ISAPI / SDK` 仍处于候选增强能力，不属于 `V1.1` 必做项
