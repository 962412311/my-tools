# 抓斗作业引导及盘存系统 — 前端 UI 设计需求文档

> 本文档面向 UI 设计师/AI 设计 Agent，完整描述系统所有界面、交互流程和功能要素。
> 技术栈：Vue 3 + Element Plus + ECharts + Three.js，需支持桌面端（1280px+）与平板端（768px+）。

---

## 一、系统概述

### 1.1 产品定位

面向工业料场（港口/钢铁/矿山）的**抓斗起重机作业引导与库存盘存一体化管控平台**。核心用户为现场操作员、远程监控员和管理人员。

### 1.2 核心能力

| 能力域 | 说明 |
|--------|------|
| 实时监控 | 视频流（HLS）、3D 点云（LiDAR）、货场 2D 俯视图联动 |
| 库存盘存 | 基于点云体积计算、多维度报表、趋势对比 |
| 远程操作 | PLC 控制链路下发、远程行车控制、补扫任务 |
| 历史追溯 | 作业历史查询、视频/点云回放、数据对比 |
| 系统管理 | 运行参数配置、料堆管理、用户权限、功能开关 |

### 1.3 用户角色

| 角色 | 权限范围 |
|------|----------|
| super_admin（超级管理员） | 全部页面 + 功能开关 |
| admin（管理员） | 全部业务页面 + 配置管理 |
| operator（操作员） | 监控、盘存、历史、回放、远程操作 |
| observer（观察员） | 监控、盘存、历史（只读） |

### 1.4 主题系统

支持三套主题，用户可在顶栏切换：
- **浅色模式**（Light）
- **深色模式**（Dark）
- **跟随系统**（Auto）

**侧边栏特性：**
- 可折叠/展开，折叠后仅显示图标
- 折叠状态持久化到 localStorage
- 移动端（<768px）侧边栏变为抽屉覆盖模式，带半透明遮罩
- 根据用户角色动态过滤菜单项

**顶栏特性：**
- 面包屑导航显示当前位置
- 右侧状态指示器组（StatusCluster）包含：主题切换、连接状态、系统状态、用户菜单
- 连接状态：显示 WebSocket/数据库 连接状态（绿色/红色指示灯）
- 系统状态：运行中/警告/错误/初始化
- 用户菜单：头像 + 用户名 + 下拉菜单（管理员显示“用户与权限 / 配置中心”，所有角色可退出登录）

### 3.3 实时监控（MonitorView）

**路由：** `/monitor`
**权限：** 所有已登录用户

**这是系统最核心、最复杂的页面。**

#### A. PageHeader
- eyebrow: "实时监控"
- title: "作业监控与现场处置"
- meta tags: 视图模式、录像状态、智能标注状态、当前时间
- actions: 录像/停止录像、截图、高级搜索

#### B. WorkbenchToolbar
- **搜索区：** 搜索框 + 搜索按钮
- **视图切换：** 分屏 / 视频 / 点云 / 货场（RadioButton 组）
- **快捷操作：** 批量盘点、视角预设、边界导入导出、智能标注开关

#### C. 主内容区（左右分栏 1.7:0.85）

**左栏 — 主视口区（根据视图模式切换）：**

**模式1: 分屏（split）**
- 上半部分：视频监控
  - 视频播放器（HLS 流，16:9）
  - 叠加信息：摄像头名称、时间戳
  - 智能标注层（最高点 Z 值、料堆名称）
  - 云台控制面板（PTZControls）：方向按钮、缩放滑块、预置位选择
- 下半部分：点云显示
  - Three.js 3D 点云渲染（Canvas）
  - 叠加信息：原始点数、显示点数、体素大小、帧率、来源
  - 控制面板：点云来源切换（最新单帧/累计地图）、显示设置（网格/坐标轴/最高点/轨迹）、点大小滑块

**模式2: 视频主视图（video）**
- 全宽视频播放器 + 智能标注

**模式3: 点云主视图（pointcloud）**
- 全宽点云渲染 + 控制面板

**模式4: 货场主视图（yard）**
- YardView 2D 俯视图组件（Canvas 绘制）
- 显示：料堆区域、行车位置、最高/最低点、历史轨迹
- 可切换：显示最高点、最低点、行车位置、轨迹

**搜索结果区（条件显示）：**
- 时间线列表展示搜索结果

**右栏 — 侧边面板（上下堆叠的 SectionCard）：**

| 面板 | 内容 |
|------|------|
| 运行态概览 | 2x2 网格状态项：大车X、小车Y、转盘θ、最高点Z、系统时间 |
| 设备诊断 | DiagnosticsPanel 组件：处理诊断信息、补扫操作按钮 |
| 最高点与处置 | 相对坐标(G系) 和 绝对坐标(W系) 的 X/Y/Z 值；复制坐标按钮；写入 PLC 按钮（危险操作，需 remote:operate 权限） |
| 实时日志 | 日志筛选输入框 + 清空按钮；日志列表（带颜色编码：info/success/warning/error） |

#### D. 弹窗列表

| 弹窗 | 宽度 | 内容 |
|------|------|------|
| 高级搜索 | 600px | 搜索类型切换（历史点云/料堆/时间段/坐标范围）；对应表单字段 |
| 批量盘点 | 800px | 料堆选择（Checkbox 组）；盘点模式（自动/手动）；算法预设选择；导出格式勾选 |
| 边界导入/导出 | 700px | Tab 切换：导入（文件上传 + 名称 + 料堆 + 坐标系）/ 导出（选择边界 + 格式 + 包含信息） |
| 视角保存/恢复 | 600px | Tab 切换：保存（名称 + 描述 + 保存选项）/ 恢复（预设列表表格 + 恢复/删除按钮） |

## 七、设计交付要求

### 7.1 必须交付

1. **所有页面的高保真设计稿**（浅色 + 深色各一套）
2. **设计 Token 文件**：颜色体系、字体、间距、圆角、阴影
3. **组件规范**：SectionCard、PageHeader、WorkbenchToolbar、StatusChip 等通用组件的状态变体
4. **弹窗规范**：所有 Dialog 的尺寸、表单布局、按钮排列
5. **数据可视化规范**：图表配色方案（与主题联动）、Tooltip 样式
6. **3D 点云/2D 货场视口**：信息叠加层的设计规范
7. **响应式适配方案**：至少 3 个关键断点的布局变体

### 7.2 重点页面优先级

| 优先级 | 页面 | 原因 |
|--------|------|------|
| P0 | 实时监控 | 系统核心页面，最复杂 |
| P0 | 首页概览 | 入口页面，信息密度高 |
| P1 | 盘存报表 | 核心业务页面 |
| P1 | 登录页 | 系统门面 |
| P2 | 远程操作 | 控制安全要求高 |
| P2 | 配置管理 | 管理员使用 |
| P3 | 历史记录/回放 | 查询类页面 |
| P3 | 功能开关/用户管理 | 低频管理页面 |

### 7.3 特殊设计注意点

1. **工业场景适配**：操作员可能戴手套操作，按钮/控件需要足够大
2. **暗光环境**：监控室通常是暗光环境，深色主题需要重点优化
3. **实时数据刷新**：状态变化需要有视觉反馈但不能过于闪烁
4. **危险操作突出**：PLC 写入等危险操作需要醒目的视觉区分
5. **信息密度平衡**：监控页信息量极大，需要层次分明不混乱
6. **3D 视口融合**：点云/视频等深色背景区域需要与页面整体风格协调
7. **中文排版**：所有界面均为中文，需要考虑中文排版的行高、字间距
