# 点云盲区补偿慢速扫描验收标准

## 目标

把“盲区补偿参数需要现场再看”收敛成固定的慢速扫描验收标准，避免每次联调都重新定义口径。

固定入口：

- `rtk bash scripts/arm/verify_blind_zone_workflow.sh`
- [`现场联调盲区补偿参数试验记录模板`](现场联调盲区补偿参数试验记录模板.md)
- [`低置信度主动补扫 README`](低置信度主动补扫README.md)
- [`点云算法设计 README`](点云算法设计README.md)

## 适用范围

- 当前项目固定的垂直朝下雷达方案
- 中心盲区 + 环形可视域 + 慢速扫描覆盖
- 当前参数主线：
  - `processing/blind_zone_annulus_thickness_factor`
  - `processing/blind_zone_height_quantile`

## 一、先过结构化体检

进入慢速扫描调参前，先要求专项脚本返回 `PASS`：

```bash
rtk bash scripts/arm/verify_blind_zone_workflow.sh
```

脚本至少应确认：

- `/api/rescan/status` 与 `/api/rescan/analyze` 可访问
- `analysisRevision / updatedAt` 已更新，不是空壳返回
- `blindZoneAnnulusThicknessFactor / blindZoneHeightQuantile` 与运行中配置一致
- `volumeBlindZoneCoverageRatio / volumeBlindZoneDensityRatio / volumeBlindZoneSupportRatio` 都是合法数值
- `blindZoneRadius` 合法，且可跟随当前位姿变化

如果这里不过，不进入参数结论阶段。

## 二、参数试探范围

当前现场经验范围先固定为：

- `processing/blind_zone_annulus_thickness_factor`：`2.0 ~ 4.0`
- `processing/blind_zone_height_quantile`：`0.20 ~ 0.35`

默认起点建议：

- `annulusThicknessFactor = 3.0`
- `heightQuantile = 0.25`

不要脱离这个范围一开始就激进放大；先证明当前扫描覆盖是稳定的，再谈扩范围。

## 三、判定时优先看什么

不要只盯最后体积值，固定按下面顺序判断：

1. `volumeBlindZoneSupportRatio`
2. `volumeBlindZoneCoverageRatio`
3. `volumeBlindZoneDensityRatio`
4. `rescanSuggested / rescanSuggestedReason`
5. `volumeSource`

解释：

- `supportRatio` 先回答“当前补偿支撑够不够”
- `coverageRatio` 再回答“扫描覆盖扇区够不够”
- `densityRatio` 再回答“已覆盖扇区的样本密度够不够”
- `rescanSuggestedReason` 用来区分是盲区支撑弱，还是扫描覆盖本身有问题
- `volumeSource` 若频繁退回 `frame`，先排查融合或轨迹窗口，不要直接调补偿参数

## 四、通过 / 复测 / 判废口径

### 通过

满足下面条件时，可把当前参数组记为“通过候选”：

- `verify_blind_zone_workflow.sh` 返回 `PASS`
- 慢速扫描稳定窗口内，`volumeBlindZoneSupportRatio` 最低值不低于 `0.35`
- `rescanSuggestedReason` 没有长期停留在 `blind_zone_support_low`
- `volumeSource` 以 `surface-map` 为主，不是频繁退回 `frame`
- 现场观察体积曲线平滑，没有明显虚高，也没有长期中心空洞

### 复测

满足下面任一情况时，标记为“复测”而不是直接通过：

- `volumeBlindZoneSupportRatio` 落在 `0.20 ~ 0.35`
- `blind_zone_support_low` 或 `ring_visibility_low` 间歇出现
- 环厚放大后支撑率提升，但体积波动没有明显改善
- 体积总体可用，但偶发抖高或偶发空洞仍存在

### 判废

满足下面任一情况时，当前参数组直接判废：

- `verify_blind_zone_workflow.sh` 结构化检查不过
- `volumeBlindZoneSupportRatio` 长时间接近 `0`
- `blind_zone_support_low` 持续拉起，且慢速扫描覆盖已经确认正常
- 体积曲线明显虚高，或中心盲区长期空洞无法恢复

## 五、调参顺序

### 先看覆盖，再看参数

如果 `coverageRatio` 长期偏低：

- 优先检查慢速扫描轨迹
- 优先检查环形可视域是否真实扫到目标区域
- 不要先把 `heightQuantile` 往上抬

### 再看密度

如果 `coverageRatio` 还可以，但 `densityRatio` 偏低：

- 先小步调整 `annulusThicknessFactor`
- 再看 `supportRatio` 是否改善

### 最后才动分位数

如果覆盖和密度都不差，但体积仍明显低估：

- 再尝试小步提高 `heightQuantile`
- 一旦体积开始抖高，先回退 `heightQuantile`

## 六、回写要求

每轮参数试验至少要回写：

- 参数矩阵
- `support / coverage / density` 的最小值、平均值、最大值
- `rescanSuggestedReason`
- `volumeSource`
- 对“体积曲线是否平滑 / 是否虚高 / 是否长期空洞”的现场结论

统一记录到 [`现场联调盲区补偿参数试验记录模板`](现场联调盲区补偿参数试验记录模板.md)。
