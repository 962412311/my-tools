## 1. 系统概述

### 1.1 系统简介

抓斗作业引导及盘存系统是专为港口、码头、仓库等物料堆场设计的智能化管理系统。系统通过激光雷达扫描、PLC控制、视频监控等技术，实现以下核心功能：

- **实时三维可视化监控**：料堆点云数据实时显示，操作员可直观看到料堆形态。
- **智能抓斗引导**：自动计算最佳抓料点（最高点），引导抓斗精准定位，避免抓空。
- **精准库存管理**：自动计算料堆体积，生成库存报表，误差≤±3%。
- **完整操作记录**：所有操作可追溯，支持历史回放和审计。
- **多维度统计分析**：支持数据导出，辅助决策。

## 2. 快速入门（新手向导）

如果您是第一次使用本系统，请跟随以下向导，5分钟内掌握核心操作。

### 2.1 登录系统

1. 打开浏览器（推荐Chrome或Edge），输入地址：`http://服务器IP:3333`
2. 输入管理员提供的用户名和密码（例如：admin / Admin@123）
3. 登录成功后默认进入【首页概览】；如果登录前是因权限跳转到登录页，登录后会返回原目标页面。
4. 管理员如需调整配置或用户权限，可从左侧导航进入【配置管理】，或通过右上角用户菜单进入【配置中心】/【用户与权限】。

### 2.2 查看实时监控

1. 登录后默认进入【首页概览】，点击左侧导航【实时监控】进入监控页面。
2. 左侧为视频画面，右侧为三维点云。
3. 尝试用鼠标拖拽旋转视角，滚动滚轮缩放。
4. 观察顶部状态区和当前页面诊断提示，确认系统处于正常状态。

### 2.3 执行一次简单盘点

1. 点击左侧导航【📦 盘存报表】进入盘存工作台。
2. 在料堆列表中找到状态为“待盘点”的料堆，点击【盘点】。
3. 保持默认参数，点击【开始扫描】。
4. 等待约30-60秒，查看结果并点击【保存】。

### 2.4 查看历史记录

1. 点击【📜 历史记录】。
2. 选择时间范围“今日”，点击【查询】。
3. 查看操作记录，点击某条记录的【详情】了解更多。

### 2.5 退出系统

1. 点击右上角用户头像。
2. 选择【退出登录】。

> **恭喜！您已掌握系统基础操作。如需深入了解，请继续阅读后续章节。**

## 3. 系统启动与登录

### 3.1 系统启动前检查

每日首次使用前，请按以下清单检查硬件和网络：

**硬件检查清单（操作员）：**

| 设备 | 检查项 | 正常状态 |
|------|--------|----------|
| 激光雷达 | 电源指示灯 | 绿色常亮 |
| 激光雷达 | 镜头 | 无遮挡、无严重积尘 |
| PLC控制器 | 电源灯 | 绿色常亮 |
| PLC控制器 | RUN灯 | 闪烁（表示运行正常） |
| 摄像头 | 镜头 | 无遮挡，画面清晰 |
| 交换机 | 对应端口指示灯 | 绿色常亮或闪烁 |
| 边缘服务器 | 电源灯 | 绿色常亮 |

**网络检查（管理员）：**
```bash
# 在任意终端执行：
ping 服务器IP地址   # 例如 ping 192.168.1.100
```

### 3.2 登录系统

#### 3.2.1 访问地址

### 4.5 信息面板解读

屏幕底部的信息面板实时更新以下内容：

**料堆信息区块：**
- **当前料堆**：系统根据抓斗位置自动识别的料堆，或手动选中的料堆。
- **当前体积**：最近一次盘点的体积（若当天未盘点，显示上次数据）。
- **最高点**：当前扫描到的最高点高度（实时更新）。
- **最低点**：当前扫描到的最低点高度（通常为地面）。
- **占地面积**：料堆底面积（基于边界计算）。

**龙门吊信息区块：**
- **位置**：大车X、小车Y、起升Z（单位米）。
- **速度**：当前运行速度（m/s）。
- **负载**：抓斗内物料重量（吨，需连接称重设备）。
- **状态**：运行中/停止/抓斗开/抓斗闭。

**引导信息区块（抓斗靠近时显示）：**
- **目标点**：系统推荐的最佳抓料点坐标。
- **距离**：当前位置到目标点的直线距离。
- **建议操作**：如“大车前进”、“小车右移”、“起升下降”等。

### 4.7 抓斗引导模式详解

当抓斗移动到料堆附近（距离<10米）时，系统自动进入引导模式，界面变化如下：

- 料堆最高点显示为**红色球体**。
- 从当前位置到目标点绘制**绿色引导线**。
- 信息面板显示详细的引导信息和操作建议。
- 如果偏离目标，建议操作会动态更新。

**操作员应遵循的步骤：**
1. 观察红色目标点和引导线。
2. 按照建议操作移动抓斗（如“大车前进”）。
3. 当距离<1米时减速慢行。
4. 到达目标点后，开始抓取作业。
5. 抓取完成后，系统自动更新料堆形态并重新计算最高点。

> **注意**：引导模式仅为辅助，操作员仍需注意现场安全。

## 5. 盘存报表与盘点操作

### 5.1 盘存工作台概述

盘存工作台用于定期盘点料堆体积，掌握库存变化。系统支持单料堆盘点、批量盘点、边界编辑和报表生成。

**盘存周期建议：**
- 高价值物料（如铁矿石）：每日盘点。
- 常规物料（如煤炭）：每周盘点。
- 低价值物料（如砂石）：每月盘点。

### 5.2 进入盘存工作台

- 点击左侧导航菜单【📦 盘存报表】。
- 页面加载后显示料堆列表和统计概览。

### 5.7 边界管理

#### 5.7.1 什么是边界

边界是料堆在地面上的轮廓线，用于：
- 限定体积计算的范围。
- 区分相邻料堆（防止重叠）。
- 剔除地面点和外部点。

#### 5.7.2 进入边界编辑

**方法1：** 在料堆详情页点击【编辑边界】。
**方法2：** 在盘点结果页点击【编辑边界】。

## 9. 坐标校准操作

### 9.1 什么是坐标校准

坐标校准是建立PLC原始坐标与世界坐标（料场实际位置）之间变换关系的过程。只有准确校准，系统才能将雷达扫描的点云正确放置在世界坐标系中，从而准确计算体积和引导抓斗。

### 9.2 校准原理

```
世界坐标 = (PLC坐标 - PLC原点) × 比例系数 × 旋转矩阵 + 世界原点
```

- **平移**：减去PLC原点，加上世界原点。
- **缩放**：比例系数（通常PLC坐标单位是毫米，需转为米）。
- **旋转**：如果PLC坐标轴方向与世界坐标系不一致，需旋转。

### 9.3 校准前准备

**必备条件：**
- [ ] 激光雷达已安装并连接，点云显示正常。
- [ ] PLC通讯正常，能实时读取坐标。
- [ ] 已知至少3个参考点的世界坐标（用全站仪或RTK测量）。
- [ ] 行车可正常移动，操作员配合。

**参考点选择原则：**
- 分布在整个料场范围（避免集中在同一区域）。
- 易于识别的位置（如立柱、墙角、固定标记）。
- 至少3个，建议4-6个点以提高精度。

### 9.4 四步校准流程

#### 步骤1：读取PLC坐标

1. 通知行车司机将抓斗移动到第一个参考点（如A点）。
2. 确保抓斗稳定停在该点正上方。
3. 在系统【配置管理 → 业务配置 → 校准流程】页面，点击【读取PLC坐标】。
4. 系统显示当前PLC坐标值（X, Y, Z）。
5. 记录下来（或系统自动暂存）。

#### 步骤2：输入世界坐标

1. 在对应的输入框中输入该参考点的世界坐标（X, Y, Z）。
2. 点击【添加参考点】。
3. 参考点列表中增加一条记录。

#### 步骤3：重复采集

重复步骤1-2，采集所有参考点。

```
参考点数据示例：
┌──────┬─────────┬─────────┬─────────┬─────────┬─────────┐
│ 点号 │ PLC X   │ PLC Y   │ 世界 X  │ 世界 Y  │ 世界 Z  │
├──────┼─────────┼─────────┼─────────┼─────────┼─────────┤
│ 1    │ 1250.00 │ 850.00  │ 50.00   │ 30.00   │ 0.00    │
│ 2    │ 3250.00 │ 1850.00 │ 150.00  │ 80.00   │ 0.00    │
│ 3    │ 2250.00 │ 1350.00 │ 100.00  │ 55.00   │ 0.00    │
│ 4    │ 1250.00 │ 1850.00 │ 50.00   │ 80.00   │ 0.00    │
└──────┴─────────┴─────────┴─────────┴─────────┴─────────┘
```

#### 步骤4：计算并验证

1. 确认参考点数量≥3，点击【计算变换矩阵】。
2. 系统自动计算并显示结果（旋转角度、比例系数、误差）。
3. **验证**：移动行车到另一个已知点（未参与计算的验证点），查看系统显示的转换后坐标是否与实际一致。误差应<0.1m。
4. 如果误差过大，检查参考点数据是否有误，或删除误差大的点重新计算。
5. 确认无误后，点击【保存校准配置】。

### 9.5 校准结果解读

```
计算结果显示：
┌─────────────────────────────────────────┐
│ 变换矩阵计算结果                        │
├─────────────────────────────────────────┤
│ 旋转角度：  15.5°                       │
│ X比例系数： 0.0500                       │
│ Y比例系数： 0.0500                       │
│ Z比例系数： 1.0000                       │
│                                         │
│ 拟合误差：  0.02m (平均)                 │
│ 最大误差：  0.05m                        │
│ 计算状态：  ✓ 成功                       │
└─────────────────────────────────────────┘
```

**误差说明：**
- 平均误差 < 0.05m：精度优秀。
- 平均误差 0.05-0.1m：可接受。
- 平均误差 > 0.1m：建议重新采集点或检查测量数据。

### 9.6 手动微调

如果自动计算后仍有微小偏差，可以手动微调参数：

| 参数 | 说明 | 调整方法 |
|------|------|----------|
| X原点偏移 | 整体X方向平移 | 正数向东移动，负数向西 |
| Y原点偏移 | 整体Y方向平移 | 正数向北移动，负数向南 |
| 旋转角度 | 整体旋转 | 正数逆时针，负数顺时针 |
| X比例 | X方向缩放 | 微调（如0.0498） |
| Y比例 | Y方向缩放 | 微调 |

**调整后**，再次验证坐标。

### 9.7 校准后的日常验证

- **每日首次使用**：查看抓斗位置是否与实际一致。
- **发现偏差**：用单个参考点快速验证，如偏差小可手动微调，如偏差大需重新校准。

**需要重新校准的情况：**
- 雷达位置发生移动。
- PLC坐标系统参数变更。
- 发现系统性偏差（如所有点都偏东0.5m）。

### 11.2 状态区说明

当前版本优先通过顶部状态区和页面内诊断区反馈系统状态：
- **连接状态**：网络、WebSocket 和基础连通性。
- **系统状态**：整体运行状态和关键告警。
- **页面内诊断**：监控页、配置中心诊断页、录像状态页会展示更细的链路信息。

**点击或进入对应页面**可查看更详细的诊断信息和错误提示。

### 11.3 常见故障及处理

#### 11.3.1 激光雷达故障（红色）

**现象**：点云不显示或卡顿。

**排查步骤**：
1. **检查电源**：雷达顶部电源灯是否亮？不亮则检查电源线。
2. **检查网络**：在服务器上执行 `ping 192.168.111.10`（雷达IP），不通则检查网线、交换机。
3. **检查IP配置**：进入系统配置，核对雷达IP是否正确。
4. **重启雷达**：断电30秒后重新上电。
5. **重启软件**：在配置页面点击【测试连接】，若仍失败，联系技术支持。

#### 11.3.2 PLC通讯故障（红色）

**现象**：龙门吊坐标不更新，引导失效。

**排查步骤**：
1. **检查PLC状态**：观察PLC面板RUN灯是否闪烁，ERR灯是否亮。
2. **检查网络**：`ping 192.168.1.100`，不通则检查网线。
3. **检查端口**：`telnet 192.168.1.100 502`，无法连接则PLC可能未开启Modbus服务。
4. **检查配置**：确认IP、端口、寄存器地址正确。
5. **重启PLC**：若允许，重启PLC（注意安全）。
6. **重启通讯**：在配置页面点击【测试通讯】。

#### 11.3.3 数据库连接故障（红色）

**现象**：无法登录，或操作缓慢。

**排查步骤**：
1. **检查数据库服务**：在服务器上执行 `systemctl status mariadb`，若未运行则 `systemctl start mariadb`。
2. **检查磁盘空间**：`df -h`，若根分区已满，清理日志或录像。
3. **检查连接数**：可能超过最大连接数，重启数据库服务。
4. **重启服务**：`systemctl restart gogs-backend.service`。

#### 11.3.4 体积计算异常

**现象**：盘点结果明显不合理（如体积为0或过大）。

**排查步骤**：
1. **检查点云质量**：点云是否完整？有无大面积缺失？若有，检查雷达。
2. **检查边界**：边界是否准确？进入边界编辑查看。
3. **检查地面高度**：地面高度设置是否正确？可尝试在配置中心对应运行参数页调整。
4. **重新扫描**：可能是临时干扰，重扫一次。
5. **查看置信度**：若置信度<0.75，结果不可靠，需重扫。

#### 11.3.5 无法登录

**现象**：输入用户名密码后提示错误。

**排查步骤**：
| 情况 | 处理 |
|------|------|
| 忘记密码 | 联系管理员重置密码 |
| 账号锁定 | 等待15分钟自动解锁，或联系管理员 |
| 账号禁用 | 联系管理员启用 |
| 浏览器问题 | 清除缓存、换浏览器试试 |
| 网络问题 | 检查网络连接，是否能访问服务器 |

### 11.4 系统恢复

#### 11.4.1 重启服务

**重启所有服务**：
```bash
# SSH登录服务器
sudo systemctl restart gogs-backend.service
sudo systemctl restart mariadb
sudo systemctl restart nginx
```

#### 11.4.2 从备份恢复

**恢复数据库**：
```bash

```

**恢复配置文件**：
1. 按部署文档恢复后端配置文件和相关备份目录，不再通过前端“导入配置”按钮操作。
2. 当前运行目录优先核对 `/userdata/GOGS/backend/config/config.ini` 及现场约定的备份文件位置。
3. 执行 `sudo systemctl restart gogs-backend.service nginx` 重启相关服务。
4. 登录前端后进入【配置管理】核对关键参数是否恢复。

## 12. 最佳实践与技巧

### 12.1 日常操作建议

- **每日启动前**：完成硬件检查清单，确保所有设备正常。
- **班前校准验证**：用固定参考点验证坐标，如有偏差微调。
- **合理设置ROI**：ROI范围不要太大，可提高处理速度。
- **定期清理录像**：设置合适的保存天数，避免磁盘满。
- **备份配置**：修改配置后及时导出备份。

### 12.2 提高盘存精度技巧

- **选择合适时间**：避免大风、大雨、大雾天气扫描。
- **确保点云质量**：扫描前清洁雷达镜头。
- **边界准确**：定期检查并修正边界。
- **多角度扫描**：对于形状复杂的料堆，可手动从不同角度扫描并取平均。
- **置信度检查**：保存结果前查看置信度，过低则重扫。

### 12.3 抓斗引导技巧

- **观察引导线**：绿色引导线指示最佳路径。
- **接近目标减速**：距离<1米时减速，避免碰撞。
- **抓取后观察**：抓取后系统自动更新最高点，可再次引导。
- **手动干预**：如自动引导点不合理（如靠近边缘），可手动选择其他点。

### 12.4 快捷键效率提升

- **Alt + 数字**：快速切换菜单。
- **空格**：监控视图暂停/继续刷新，便于仔细观察。
- **R**：重置视角。
- **S**：截图（监控和回放通用）。

## 13. 常见问题解答（FAQ）

**Q1: 系统支持多台龙门吊同时使用吗？**
A: 当前现行版本按单台龙门吊部署和验收，多台场景需单独评估方案、接口边界和现场流程。

**Q2: 如何将体积数据导出到ERP系统？**
A: 可通过报表导出CSV，或使用API接口（需开发）。

**Q3: 忘记管理员密码怎么办？**
A: 联系软件供应商通过后台重置。

**Q4: 录像保存多久？空间不足怎么办？**
A: 默认30天，可修改配置。空间不足时系统会自动删除最旧录像。

**Q5: 点云显示很卡怎么办？**
A: 在监控视图设置中降低采样率（如设为50%），或减小点大小。

**Q6: 盘点时提示“点云质量差”是什么原因？**
A: 可能雷达镜头脏污、天气恶劣或料堆表面反射差。清洁镜头后重试。

**Q7: 坐标校准误差大怎么办？**
A: 检查参考点测量是否准确，尽量让点分布均匀。可删除误差大的点重新计算。

**Q8: 如何升级系统软件？**
A: 由供应商提供升级包，在服务器上执行升级脚本，或通过Web管理界面上传。
