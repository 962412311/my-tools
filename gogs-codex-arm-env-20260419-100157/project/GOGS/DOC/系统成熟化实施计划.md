# 系统成熟化实施计划

## 目的

这份文档把“成熟化迭代路线图”拆成可执行任务，作为后续版本推进、现场联调、文档更新和验收收口的统一入口。

默认按三个阶段推进：

1. `V1.1 稳定可用版`
2. `V1.2 生产增强版`
3. `V1.3 厂商增强版`

## 统一状态定义

后续所有成熟化任务统一使用下面三种状态：

- `已完成`：实现、验证、文档、状态口径都已收口
- `待验证`：代码或配置已落地，但缺实机或现场验收
- `未开始`：尚未进入实现

禁止把“代码已写”直接标记成“已完成”。

## 每轮推进的固定输出

每次推进成熟化任务，至少要输出：

1. 改了什么
2. 为什么这样改
3. 影响范围
4. 验证情况
5. 文档更新
6. 剩余风险

## 近期建议先执行的 10 个任务

1. 完成真实视频源截图、录像、回放的兼容性验证
2. 完成长时预览、断流、摄像机重启、`mediamtx` 重启后的恢复验证
3. 完成 ARM 目标机完整构建、部署、服务启动与验活闭环
4. 建立海康设备标准接入模板和现场 SOP
5. 在真实 PLC / 称重设备上按既有模板完成现场回写和异常诊断确认
6. 把视频自检扩展为统一的 `video / ptz / plc / scales / database / disk / process` 自检入口
7. 更新运维与验收文档，固化升级、回滚、验活和现场回写流程
8. 为关键前端交互补最小流程测试，为关键后端接口补稳定回归测试
9. 输出一版 `V1.1` 版本化验收记录，并把现场结论回写 `todo.md`
10. 在不破坏 Web 主链的前提下，继续评估海康 SDK / ISAPI 作为后端增强能力的真实收益
