## 1. 系统概述

### 1.1 系统简介

抓斗作业引导及盘存系统是专为港口、码头、仓库等物料堆场设计的智能化管理系统。系统通过激光雷达扫描、PLC控制、视频监控等技术，实现：

- **实时三维可视化监控** - 料堆点云数据实时显示
- **智能抓斗引导** - 自动计算最佳抓料点
- **精准库存管理** - 自动计算料堆体积
- **完整操作记录** - 所有操作可追溯
- **多维度统计分析** - 支持数据导出

## 2. 系统启动与登录

### 2.1 系统启动流程

#### 2.1.1 硬件设备检查

在启动系统前，请按以下顺序检查硬件设备：

**第一步：检查激光雷达**
1. 确认雷达电源指示灯为绿色常亮
2. 检查网络连接状态，确认网线已插紧
3. 观察雷达旋转机构是否正常运转
4. 确认雷达视野范围内无遮挡物

**第二步：检查PLC控制器**
1. 确认PLC电源指示灯为绿色
2. 检查RUN指示灯是否闪烁或常亮
3. 确认以太网端口指示灯正常闪烁
4. 验证急停按钮处于释放状态

**第三步：检查网络设备**
1. 确认交换机电源正常
2. 检查各端口指示灯状态
3. 测试网络连通性（ping测试）

#### 2.1.2 软件服务启动

**后端服务启动：**
```bash
# 1. 启动数据库服务
sudo systemctl start mysql

# 2. 启动后端核心服务
./backend/grab_system_server

# 3. 检查服务状态
curl http://localhost:8080/api/status
```

**前端服务启动：**
```bash
# 1. 进入前端目录
cd frontend

# 2. 启动开发服务器
npm run dev

# 3. 确认服务启动成功
# 控制台显示：Local: http://localhost:3333/
```

### 2.2 系统登录详解

#### 2.2.1 登录前准备

1. **浏览器要求**
   - 推荐使用 Chrome 90+ 或 Edge 90+
   - 最低分辨率要求：1920×1080
   - 需要启用 JavaScript 和 WebGL

2. **网络环境**
   - 确保客户端能访问服务器IP
   - 测试命令：`ping {服务器IP}`
   - 端口要求：3333（前端）、8080（后端API）、12345（WebSocket）

### 2.3 修改密码详解

#### 2.3.1 密码修改入口

**入口1：通过用户菜单**
1. 点击右上角用户头像
2. 在下拉菜单中选择"修改密码"
3. 弹出密码修改对话框

**入口2：通过个人设置**
1. 点击右上角设置图标
2. 选择"个人设置"
3. 切换到"安全设置"标签
4. 点击"修改密码"

#### 2.3.3 密码强度要求

| 强度等级 | 要求 | 示例 |
|----------|------|------|
| 弱 | 8位以上 | 12345678 |
| 中 | 8位以上，含字母和数字 | Admin123 |
| 强 | 8位以上，含大小写字母、数字、特殊字符 | Admin@123# |

**密码安全规范：**
- 禁止使用与用户名相同的密码
- 禁止使用连续数字（12345678）
- 禁止使用重复字符（aaaaaaaa）
- 建议包含特殊字符（!@#$%^&*）

### 2.4 系统退出详解

#### 2.4.1 正常退出

**操作步骤：**
1. 点击右上角用户头像
2. 选择"退出登录"
3. 确认退出对话框
4. 等待退出完成（清除本地缓存）

#### 2.4.2 自动退出机制

| 场景 | 超时时间 | 说明 |
|------|----------|------|
| 无操作 | 30分钟 | 鼠标键盘无操作 |
| 页面隐藏 | 10分钟 | 切换至其他标签页 |
| 网络断开 | 立即 | WebSocket连接断开 |

#### 2.4.3 退出后清理

系统自动清理：
- 本地存储的Token
- 临时缓存数据
- WebSocket连接
- 定时刷新任务

### 5.3 新建盘点深度操作

#### 5.3.1 盘点前准备

**检查清单：**
- [ ] 激光雷达运行正常
- [ ] PLC通讯正常
- [ ] 料堆区域无作业车辆
- [ ] 天气条件良好（无大雾、暴雨）
- [ ] 系统时间准确
- [ ] 坐标校准已完成且准确

**体积计算原理说明：**

系统体积计算基于PCL（Point Cloud Library）点云处理库，采用以下算法：

```
体积计算流程：
1. 点云采集 → 2. 滤波处理 → 3. 地面分割 → 4. 边界检测 → 5. 体积计算

算法说明：
- 凸包算法：计算料堆底面多边形
- 平均高度法：底面积 × 平均高度 = 体积
- 体素网格：对点云进行体素化降采样
```

**关键参数说明：**

| 参数名 | 默认值 | 说明 | 影响 |
|--------|--------|------|------|
| voxel_size | 0.05m | 体素大小 | 值越小精度越高但计算量越大 |
| roi_width | 10.0m | ROI区域宽度 | 应大于料堆最大宽度 |
| roi_height | 10.0m | ROI区域高度 | 应大于料堆最大高度 |
| statistical_outlier_k | 50 | 统计滤波邻域点数 | 值越大滤波越平滑 |
| statistical_outlier_std | 1.0 | 标准差阈值 | 值越小过滤越严格 |
| ground_fitting | histogram | 地面拟合方法 | histogram/平面拟合 |

### 8.1 配置管理概述

系统配置分为以下模块：
- 硬件配置：激光雷达、PLC、摄像头
- 网络配置：IP地址、端口、协议
- 业务参数：扫描间隔、体积阈值
- 告警设置：告警级别、通知方式
- 数据备份：备份策略、存储位置

**配置修改权限：**
| 配置项 | 超级管理员 | 管理员 | 操作员 |
|--------|------------|--------|--------|
| 硬件配置 | ✓ | ✓ | ✗ |
| 网络配置 | ✓ | ✗ | ✗ |
| 业务参数 | ✓ | ✓ | ✗ |
| 告警设置 | ✓ | ✓ | ✗ |
| 数据备份 | ✓ | ✓ | ✗ |

**配置参数说明：**

| 参数项 | 默认值 | 配置说明 | 注意事项 |
|--------|--------|----------|----------|
| IP地址 | 192.168.111.10 | 激光雷达的网络地址 | 必须与雷达实际IP一致，修改后需重启服务 |
| 数据端口 | 5600 | 点云数据接收端口 | 确保防火墙放行此端口 |

**配置步骤：**

1. **确认雷达IP地址**
   ```bash
   # 在服务器上测试雷达连通性
   ping 192.168.111.10
   
   # 测试雷达数据端口
   telnet 192.168.111.10 5600
   ```

2. **修改配置参数**
   - 登录系统管理界面
   - 进入"配置管理" → "通信配置"标签
   - 修改雷达IP地址和端口
   - 点击"保存配置"

3. **验证配置生效**
   - 点击"测试连接"按钮
   - 查看连接状态指示灯
   - 观察点云数据是否正常显示

4. **配置备份**
   - 修改前导出当前配置
   - 修改后记录变更日志
   - 保留配置版本历史

**PLC通讯数据格式：**

根据代码实现，PLC通讯使用Modbus协议，数据寄存器映射如下：

**输入寄存器（PLC→系统）：**
| 寄存器地址 | 数据类型 | 说明 |
|------------|----------|------|
| 0-1 | Float32 | 大车位置X（单位：米） |
| 2-3 | Float32 | 小车位置Y（单位：米） |
| 4-5 | Float32 | 转盘角度θ（单位：弧度） |
| 6 | UInt16 | 设备状态码 |
| 7-8 | UInt32 | 时间戳（毫秒） |

**输出寄存器（系统→PLC）：**
| 寄存器地址 | 数据类型 | 说明 |
|------------|----------|------|
| 100-101 | Float32 | 最高点相对X坐标 |
| 102-103 | Float32 | 最高点相对Y坐标 |
| 104-105 | Float32 | 最高点相对Z坐标 |
| 106-107 | Float32 | 最高点绝对X坐标 |
| 108-109 | Float32 | 最高点绝对Y坐标 |
| 110-111 | Float32 | 最高点绝对Z坐标 |
| 112 | UInt16 | 心跳计数器 |
| 113 | UInt16 | 系统状态码 |

**配置步骤：**

1. **确认PLC网络参数**
   ```bash
   # 测试PLC连通性
   ping 192.168.1.100
   
   # 测试Modbus端口
   telnet 192.168.1.100 502
   ```

2. **配置PLC通讯参数**
   - 登录系统管理界面
   - 进入"配置管理" → "通信配置"标签
   - 选择通讯协议（TCP或RTU）
   - 填写PLC的IP地址和端口
   - 设置从站地址（默认1）

3. **设置通讯周期**
   - 读取周期：50ms（20Hz），用于实时获取龙门吊位置
   - 写入周期：100ms（10Hz），用于向PLC发送最高点坐标
   - 响应超时：500ms，超过此时间视为通讯失败

4. **测试通讯连接**
   - 点击"测试通讯"按钮
   - 观察状态指示灯变化
   - 查看实时数据是否正常刷新

5. **验证数据准确性**
   - 对比PLC显示的位置与系统显示
   - 检查坐标方向是否一致
   - 确认单位换算正确（毫米/米）

**故障排查：**

| 故障现象 | 可能原因 | 解决方法 |
|----------|----------|----------|
| 连接失败 | IP地址错误 | 检查PLC实际IP地址 |
| 连接失败 | 端口被占用 | 检查502端口是否被防火墙阻挡 |
| 数据异常 | 寄存器地址错误 | 核对PLC程序中的寄存器映射 |
| 数据不刷新 | 通讯周期过长 | 减小读取周期值 |
| 频繁断线 | 网络不稳定 | 检查网线连接，增大超时时间 |

### 8.3 坐标校准深度操作

坐标校准是确保系统准确获取龙门吊位置和计算料堆体积的关键步骤。系统通过建立PLC坐标系（原始坐标）与世界坐标系（料场实际坐标）之间的变换关系，实现坐标准确转换。

#### 8.3.1 坐标系统说明

**坐标系定义：**

| 坐标系 | 名称 | 说明 | 原点 |
|--------|------|------|------|
| L系 | 激光雷达坐标系 | 雷达自身的三维坐标系统 | 雷达安装位置 |
| G系 | 抓斗坐标系 | 相对于抓斗的坐标系统 | 抓斗中心 |
| W系 | 世界坐标系 | 料场全局坐标系统 | 料场参考点 |

**坐标变换流程：**
```
激光雷达坐标(L) → 抓斗坐标(G) → 世界坐标(W)
     ↓                    ↓              ↓
  原始扫描数据      相对位置数据      绝对位置数据
```

#### 8.3.2 校准前准备工作

**必备条件：**
- [ ] 激光雷达已正确安装并连接
- [ ] PLC通讯正常，能实时读取龙门吊位置
- [ ] 已知料场中至少3个参考点的世界坐标
- [ ] 行车可以正常操作移动
- [ ] 校准人员熟悉行车操作

**工具准备：**
- 全站仪或GPS定位设备（用于测量参考点世界坐标）
- 卷尺（辅助测量）
- 校准记录表

**安全注意事项：**
1. 校准过程中确保料场无其他作业
2. 行车移动时注意安全距离
3. 操作人员与行车司机保持通讯畅通
4. 恶劣天气（大风、暴雨）禁止校准

**步骤1：读取PLC坐标**

操作目的：获取行车在料场中某个已知位置的PLC原始坐标值。

详细操作步骤：

1. **选择参考点**
   - 在料场中选择第一个参考点（建议选择料场角落或明显标记点）
   - 确保该点的世界坐标已知（通过全站仪测量）
   - 记录参考点编号：参考点1

2. **操作行车到位**
   ```
   操作指令：
   - 通知行车司机将抓斗移动到参考点1正上方
   - 抓斗高度建议：离地面2-3米
   - 确认行车已停止移动并稳定
   ```

3. **读取PLC坐标**
   ```
   系统操作：
   1. 在坐标校准界面，确认当前步骤为"步骤1: 读取PLC坐标"
   2. 点击【读取PLC坐标】按钮
   3. 等待系统读取（约1-2秒）
   4. 记录显示的PLC坐标值
   ```

4. **记录数据**
   ```
   参考点1数据记录：
   ┌──────────┬──────────┬──────────┐
   │   PLC X   │   PLC Y   │   PLC Z   │
   ├──────────┼──────────┼──────────┤
   │  1250.00 │   850.00 │   120.00 │
   └──────────┴──────────┴──────────┘
   ```

5. **验证数据合理性**
   - 检查X、Y值是否在料场范围内（如0-500米）
   - 检查Z值是否为正值（高度）
   - 如数据异常，重新读取或检查PLC连接

6. **进入下一步**
   - 确认数据正常后，系统自动进入步骤2
   - 或点击【下一步】按钮手动进入

**步骤3：计算变换矩阵**

操作目的：系统根据采集的参考点数据，计算PLC坐标到世界坐标的变换矩阵。

详细操作步骤：

1. **检查参考点数量**
   ```
   系统检查：
   - 如参考点少于3个，提示"至少需要3个参考点"
   - 按钮【计算变换矩阵】处于禁用状态
   - 需返回步骤1继续采集
   ```

2. **执行计算**
   ```
   系统操作：
   1. 确认参考点列表中数据正确
   2. 点击【计算变换矩阵】按钮
   3. 等待计算完成（约1-3秒）
   ```

3. **理解变换原理**
   ```
   坐标变换公式：
   
   世界坐标 = (PLC坐标 - PLC原点) × 旋转矩阵 × 比例系数 + 世界原点
   
   其中：
   - 旋转矩阵：根据旋转角度计算
   - 比例系数：PLC坐标单位与世界坐标单位的比值
   - 原点偏移：两个坐标系原点的差值
   
   代码实现（简化）：
   angle = rotationAngle × π / 180
   cos = cos(angle)
   sin = sin(angle)
   
   dx = plcX - plcOriginX
   dy = plcY - plcOriginY
   
   worldX = (dx × cos - dy × sin) × scaleX + worldOriginX
   worldY = (dx × sin + dy × cos) × scaleY + worldOriginY
   worldZ = (plcZ - plcOriginZ) × scaleZ + worldOriginZ
   ```

4. **查看计算结果**
   ```
   计算结果显示：
   ┌─────────────────────────────────────────┐
   │ 变换矩阵计算结果                        │
   ├─────────────────────────────────────────┤
   │ 旋转角度：  15.5°                       │
   │ X比例系数： 0.0500                      │
   │ Y比例系数： 0.0500                      │
   │ Z比例系数： 1.0000                      │
   │                                         │
   │ 拟合误差：  0.02m (平均)                │
   │ 最大误差：  0.05m                       │
   │ 计算状态：  ✓ 成功                      │
   └─────────────────────────────────────────┘
   
   误差说明：
   - 平均误差 < 0.05m：精度良好
   - 平均误差 0.05-0.1m：精度一般，可接受
   - 平均误差 > 0.1m：精度较差，建议重新校准
   ```

5. **评估校准质量**
   - 如误差过大，可能原因：
     - 参考点测量不准确
     - 行车未完全停止就读取
     - 参考点分布不合理（应在料场四周均匀分布）
   - 建议删除误差大的参考点，重新采集

6. **进入下一步**
   - 确认计算结果可接受
   - 点击【下一步】进入步骤4

**步骤4：验证校准结果**

操作目的：通过移动行车到不同位置，验证坐标转换的准确性。

详细操作步骤：

1. **选择验证点**
   ```
   验证点选择：
   - 选择1-2个未参与校准的已知点
   - 或选择料场中明显的位置（如料堆中心）
   ```

2. **移动行车到验证点**
   ```
   操作指令：
   - 通知行车司机将抓斗移动到验证点
   - 确保行车稳定停止
   ```

3. **查看坐标对比**

#### 8.4.1 点云处理参数详解

**体素大小 (voxel_size)**

| 设置值 | 适用场景 | 精度 | 性能 |
|--------|----------|------|------|
| 0.01m | 高精度要求、小料堆 | 高 | 低 |
| 0.05m | 标准场景（推荐） | 中 | 中 |
| 0.10m | 大料堆、快速估算 | 低 | 高 |
| 0.20m | 快速预览 | 很低 | 很高 |

配置建议：
- 常规盘点：0.05m
- 精细测量：0.02-0.03m
- 快速扫描：0.10m

**ROI（感兴趣区域）设置**

ROI区域定义了点云处理的空间范围，超出此范围的点将被过滤。

```
ROI设置原则：
┌─────────────────────────────────────────┐
│                                         │
│    ROI区域应完全包含目标料堆            │
│    并留有一定余量（建议2-5米）          │
│                                         │
│    ┌─────────────┐                      │
│    │             │                      │
│    │   料堆      │ ← 被测料堆          │
│    │             │                      │
│    └─────────────┘                      │
│                                         │
│    ROI边界应在此处                      │
│    （料堆边界 + 余量）                  │
│                                         │
└─────────────────────────────────────────┘
```

**统计滤波参数**

统计滤波用于去除点云中的噪声点（离群点）。

```
滤波原理：
1. 对每个点，查找其K个最近邻点
2. 计算这K个邻点到中心点的平均距离
3. 计算所有平均距离的全局均值μ和标准差σ
4. 如果某点的平均距离 > μ + Std × σ，则判定为离群点并移除

参数设置：
- K值越大，考虑的邻域范围越大，滤波越平滑
- Std越小，过滤条件越严格，移除的点越多
```

配置建议：
- K值：30-100（默认50）
- Std：1.0-2.0（默认1.0）
- 噪声较多时：增大K值或减小Std
- 需要保留细节时：减小K值或增大Std

#### 8.4.2 体积计算参数详解

**密度系数**

密度系数用于将体积转换为重量，不同物料密度不同：

| 物料类型 | 密度范围(t/m³) | 推荐值 |
|----------|----------------|--------|
| 铁矿石 | 2.5-3.5 | 3.0 |
| 煤炭 | 1.2-1.5 | 1.4 |
| 石灰石 | 2.6-2.8 | 2.7 |
| 焦炭 | 0.4-0.6 | 0.5 |
| 砂石 | 1.6-2.0 | 1.8 |

**地面拟合方法**

| 方法 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| 直方图法 | 复杂地形、不规则地面 | 适应性强 | 计算量较大 |
| 平面拟合 | 平整地面、标准料场 | 计算快速 | 对复杂地形效果差 |

**置信度阈值**

置信度反映体积计算结果的可靠程度：
- 置信度 = 有效点数 / 总点数
- 置信度 > 0.9：结果非常可靠
- 置信度 0.75-0.9：结果可靠
- 置信度 0.5-0.75：结果存疑，建议重新扫描
- 置信度 < 0.5：结果不可靠，必须重新扫描

### 10.2 常见故障处理

#### 10.2.1 激光雷达故障

**故障现象：** 雷达状态显示红色

**排查步骤：**
1. 检查雷达电源指示灯
   - 不亮 → 检查电源线和供电
   - 红色 → 雷达内部故障，重启尝试

2. 检查网络连接
   ```bash
   ping 192.168.1.100
   ```
   - 不通 → 检查网线和交换机
   - 通但丢包 → 检查网络质量

3. 检查雷达视野
   - 确认无遮挡物
   - 清洁雷达镜头

4. 重启雷达
   - 断电等待30秒
   - 重新上电
   - 等待启动完成（约2分钟）

#### 10.2.2 PLC通讯故障

**故障现象：** PLC状态显示红色

**排查步骤：**
1. 检查PLC电源和运行状态
   - RUN灯应绿色常亮或闪烁
   - ERR灯应熄灭

2. 检查网络连接
   ```bash
   ping 192.168.1.10
   telnet 192.168.1.10 502
   ```

3. 检查PLC程序
   - 确认Modbus服务已启用
   - 检查寄存器地址配置

4. 重启PLC通讯
   - 在系统配置页面点击"测试通讯"
   - 或重启后端服务

#### 10.2.3 数据库连接故障

**故障现象：** 数据库状态显示红色

**排查步骤：**
1. 检查数据库服务状态
   ```bash
   sudo systemctl status mysql
   ```

2. 检查连接参数
   - 主机地址
   - 端口号
   - 用户名密码

3. 检查磁盘空间
   ```bash
   df -h
   ```

4. 重启数据库服务
   ```bash
   sudo systemctl restart mysql
   ```

### 10.3 系统恢复

**数据备份恢复：**

## 11. 安全操作规范

### 11.1 账号安全

**密码管理规范：**
1. 初始密码必须在首次登录后修改
2. 密码长度不少于8位，包含大小写字母和数字
3. 密码有效期不超过90天
4. 禁止使用与用户名相同的密码
5. 禁止在多个系统使用相同密码

**登录安全：**
1. 离开工作站必须退出登录
2. 发现异常登录立即修改密码并报告
3. 不使用公共电脑登录系统
4. 不将账号借给他人使用

### 11.2 操作安全

**关键操作确认：**
| 操作 | 确认方式 | 备注 |
|------|----------|------|
| 删除用户 | 二次确认对话框 | 不可恢复 |
| 修改系统配置 | 影响分析提示 | 可能影响运行 |
| 删除历史数据 | 密码验证+确认 | 不可恢复 |
| 修改边界 | 预览确认 | 影响体积计算 |

**操作审计：**
- 所有操作记录日志
- 关键操作发送通知
- 定期审计操作记录

### 11.3 数据安全

**数据备份策略：**
| 数据类型 | 备份频率 | 保留期限 | 存储位置 |
|----------|----------|----------|----------|
| 数据库 | 每日凌晨2点 | 30天 | 本地+远程 |
| 配置文件 | 修改后自动 | 10个版本 | 本地 |
| 录像文件 | 实时 | 30天 | 专用存储 |

**数据访问控制：**
- 敏感数据导出需要审批
- 导出文件加密存储
- 定期清理过期数据
