# V1.1 视频链路版本化验收清单

## 目标

这份清单用于把 `V1.1` 视频链路收口成一次正式版本化验收，而不是继续停留在零散联调。

适用范围：

- 海康摄像机
- 固定 `ARM Linux + Qt 后端 + Vue 前端`
- 当前视频主线：`RTSP + MediaMTX + WebRTC/WHEP + HLS + ONVIF + GStreamer`

## 一、验收前置条件

以下条件不满足时，不进入正式验收：

- 监控页顶部状态已全部可见
- `camera/main_stream_* / sub_stream_*` 已按现场基线配置
- `recording/input_codec` 已按当前机型确认
- `/api/video/self-check` 可正常返回
- `rtk bash scripts/arm/verify_field_acceptance_bundle.sh` 可执行
- `rtk bash scripts/arm/verify_browser_matrix_readiness.sh` 可执行
- 已执行 `rtk bash scripts/arm/ensure_current_acceptance_packet.sh`
- 已准备 [`DOC/当前现场验收包/视频实机验收矩阵.md`](当前现场验收包/视频实机验收矩阵.md)
- 已准备 [`DOC/当前现场验收包/现场联调验收记录.md`](当前现场验收包/现场联调验收记录.md)
- 已准备 [`视频现场回写顺序卡`](视频现场回写顺序卡.md)

## 二、版本化验收范围

本次 `V1.1` 只验收这些内容：

1. 实时预览
2. 浏览器兼容降级
3. 抓图
4. 录像
5. 回放
6. PTZ 与预置位
7. 统一视频自检

不在本次范围内：

- 海康 `SDK / ISAPI` 正式接入
- 非现场目标浏览器的泛兼容结论
- 浏览器外原生客户端链路

## 三、通过标准

### A. 实时预览

- 默认实时链路稳定优先命中 `WebRTC`
- `WebRTC` 不可用时可明确降级到 `HLS`
- 监控页顶部能直接显示链路、异常、`MediaMTX`、端口探测、配置源

### B. 浏览器兼容

- `H.264 / H.265` 实机矩阵有真实回写
- 明确哪些浏览器允许直放 `H.265`
- 明确哪些浏览器必须退回 `H.264`

### C. 抓图

- 本地截图成功，或本地失败后后端兜底成功
- `/api/video/snapshot` 失败时有真实错误留痕

### D. 录像

- `recording/status` 返回前提、健康、恢复、保留策略信息
- 录像可启动、可停止、可落盘
- 异常退出时恢复状态有留痕
- 录制文件可播放、可下载

### E. 回放

- 回放页能加载录像列表
- 录像文件能正常播放
- 录像列表筛选、排序与当前焦点一致

### F. PTZ

- `/api/video/ptz/status` 配置态完整
- `move / relative-move / stop / presets / home` 都完成实机验证
- `WS-Security` 要求明确并已记录

### G. 自检

- `/api/video/self-check` 返回 `overallStatus / summaryMessage / checkItems`
- 监控页顶部能直接显示 `视频自检 / 自检摘要`
- `rtk bash scripts/arm/verify_field_acceptance_bundle.sh` 与接口口径一致

本次验收完成后，必须同时落下这些输出物：

1. 一份已填写的 [`DOC/当前现场验收包/视频实机验收矩阵.md`](当前现场验收包/视频实机验收矩阵.md)
2. 一份已填写的 [`DOC/当前现场验收包/现场联调验收记录.md`](当前现场验收包/现场联调验收记录.md)
3. `todo.md` 中浏览器矩阵与 `V1.1` 视频链路状态更新

5. 执行 `rtk bash scripts/arm/finalize_remaining_acceptance_closure.sh`，生成正式归档与最终收口报告

- `已通过`：模板、记录、结论、todo 状态都已回写
