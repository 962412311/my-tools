# 32-security

## 目的
本文件定义开发、文档、知识管理、部署过程中的基础安全要求。

---

## 核心原则
- 密钥、token、密码不进代码
- 敏感配置不进公开文档
- 示例、日志、截图默认脱敏
- 最小暴露原则优先于方便原则

---

## 代码与配置要求
- 使用环境变量或项目约定的安全配置方式
- 提供 `.env.example` 时只保留键名与示例值
- 不在源码、脚本、测试样例中硬编码敏感信息
- 不在 commit 中包含真实密钥

---

## 文档要求
写文档时：
- 不暴露真实账号、密码、token、cookie、私有地址
- 截图、日志、示例命令默认做脱敏
- 对外可见文档与内部操作文档要区分清楚

---

## 调试与日志要求
- 调试输出中避免打印敏感字段
- 报错截图和日志分享前先检查脱敏
- 不为了排障方便而长期保留敏感调试输出

---

## 部署与环境要求
- 环境变量名称统一
- 敏感参数来源明确
- 不把生产环境配置复制到示例文件中
- 不把真实凭证写入部署脚本注释或 README

---

## 风险意识
一旦涉及以下内容，默认提高警惕：
- API keys
- Access tokens
- Passwords
- Cookies
- Database URLs
- Internal endpoints
- Private certificates
- Personal data

---

## 禁止行为
- 在代码、文档、提交中留下真实密钥
- 在截图或日志中暴露敏感信息
- 用“只是本地临时用”作为写入密钥的理由
- 将安全问题视为后续再处理的可选项
